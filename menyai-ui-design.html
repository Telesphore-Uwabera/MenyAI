Key Design Principles

  • Kinyarwanda-First: All text, navigation, and instructions are exclusively in Kinyarwanda
  • Visual Literacy: Heavy use of emojis, icons, and illustrations for users with minimal reading skills
  • Audio-First: Every lesson includes audio pronunciation and instructions
  • Optional AI: AI assistance is clearly marked and optional via dedicated button - never mandatory
  • Warm & Welcoming: Friendly gradients, rounded corners, and encouraging language
  • Clear Hierarchy: Large, readable text with clear visual separation between sections
  • Progress Visibility: Always show learners where they are and what they've accomplished

Typography

  • Primary Font: Plus Jakarta Sans (modern, friendly, highly legible)
  • Sizes:
    • Headings: 24-36px (bold)
    • Body: 14-16px (medium)
    • Small text: 11-12px (for metadata)
  • Weight: Range from 400 (regular) to 800 (extra bold) for emphasis

UI Components

  • Cards: White backgrounds with subtle shadows, 16px border radius
  • Buttons: 12px border radius, clear hover states, generous padding
  • AI Button: Purple gradient to distinguish from regular content, pulsing animation
  • Progress Bars: Green gradient fill showing completion percentage
  • Bottom Navigation: Sticky, always accessible with 4 main sections

Iconography

  • Emojis: Used extensively for visual communication (üè† üìö üìä ü§ñ etc.)
  • Purpose: Helps low-literacy users navigate by visual recognition
  • Consistency: Same emoji always means the same function

AI Assistant Features

  • Activation: Prominent purple button with robot emoji
  • Chat Interface: Familiar messaging-style layout
  • Kinyarwanda Only: AI speaks exclusively in Kinyarwanda
  • Context Aware: AI knows which lesson user is on
  • Quick Actions: Suggested prompts for common questions
  • Visual Indicators: Green dot shows AI is active/ready

Accessibility Considerations

  • Large Touch Targets: Minimum 44px height for all interactive elements
  • High Contrast: Text meets WCAG AA standards against backgrounds
  • Audio Support: Every text element can be played aloud
  • Visual Feedback: Clear hover and active states for all buttons
  • Simple Navigation: Maximum 4 main sections, clear back buttons

Animation & Interaction

  • Subtle Transitions: 0.3s ease for hover effects
  • Pulsing AI Button: Gentle animation to indicate availability
  • Card Hover: Lift effect (-4px translateY) to show interactivity
  • Progress Animation: Smooth width transition for progress bars

Mobile-First Approach

  • Screen Size: Optimized for 320px-420px width (common Android phones)
  • Vertical Scrolling: Single column layout for easy thumb navigation
  • Bottom Navigation: Thumb-friendly zone for primary actions
  • Data Efficiency: AI activation is optional to save mobile data
